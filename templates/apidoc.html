My To-Do List

Hi! Thanks for stopping by! This page is supposed to explain how my REST API actually works.

This is my first time making a REST API. Which means I'm still learning.

I learned that a REST API is an application that has the following characteristics:

  • Has a set of resources (at least one)
  • Receive structured and well-defined requests
  • Provides a structured response
  • Might require authentication in order to provide a response

My API Resources

(1) .../api/todos

GET

If you send a GET request to this path, you can see all To-Do items displayed in JSON format.

Output: Will display ALL To-Do's and their parameters.


POST

This will create a new To-Do. For this you need to include two parameters:

  • 'task': A description for the task
  • 'category': A category for this task

Output: Will display the NEW To-Do and its parameters.


(2) .../api/todos/[todo-id]

GET

Provides a JSON output of parameters for the To-Do item.

Output: Will display the requested To-Do and its parameters.



PUT

If you send a PUT request to this resource, you can update the following parameter values for the corresponding todo:

  • 'task': You can modify the task for this To-Do
  • 'completed': You can modify the completion flag. True/False
  • 'category': You can modify the category of this To-Do

Output: Will display the updated To-Do and its parameters.


DELETE

Removes the To-Do.

Output: There is no output here.
